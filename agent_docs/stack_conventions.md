# Stack Conventions
# Read this when: starting a new project, adding new features, or unsure where a file belongs.

---

## Default Stack (override in CLAUDE.md [PROJECT CONFIG] if different)
```
Framework     : Next.js 15 (App Router)
Styling       : Tailwind CSS v4
Components    : shadcn/ui
Icons         : Lucide React
Fonts         : Google Fonts via next/font
State (client): Zustand
State (server): TanStack Query
Forms         : React Hook Form + Zod
Animations    : Framer Motion
Images        : next/image (proxy all external images)
TypeScript    : strict mode
Linting       : Biome (lint + format combined)
```

---

## File Structure (Next.js App Router)
```
src/
├── app/                  # routes, layouts, pages
│   ├── (marketing)/      # route groups for landing pages
│   ├── (app)/            # route groups for authenticated app
│   └── api/              # API routes
├── components/
│   ├── ui/               # shadcn/ui base components (don't modify)
│   └── [feature]/        # feature-specific components
├── lib/                  # utilities, helpers, config
├── hooks/                # custom React hooks
└── types/                # shared TypeScript types

design-system/            # generated by UI UX Pro Max
agent_docs/               # these files — Claude reads them when needed
```

---

## Naming Conventions
```
Components    : PascalCase     → UserProfileCard.tsx
Hooks         : camelCase      → useUserProfile.ts
Utilities     : camelCase      → formatCurrency.ts
Constants     : UPPER_SNAKE    → MAX_RETRY_COUNT
CSS classes   : kebab-case     → .user-profile-card
```

---

## TypeScript Rules
- Strict mode always — no `any`, no `@ts-ignore` without a comment explaining why
- Prefer `type` over `interface` unless you need declaration merging
- Co-locate types with the feature they belong to, not in a global `types/` dump

---

## Key Commands
```bash
npm run dev          # start dev server
npx tsc --noEmit     # type check (no output files)
npx biome check --write .  # lint + format in one pass
```

---

## Don't
- Don't write custom CSS for things Tailwind can handle
- Don't duplicate component logic — extract to shared component or hook
- Don't skip TypeScript types for "quick" work
- Don't put secrets in code — use `process.env` and `.env` (gitignored)
